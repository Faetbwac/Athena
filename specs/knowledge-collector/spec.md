# 功能规范: Aegis 知识采集增强器

**项目**: Athena（个人知识库系统）
**增强器**: Aegis
**功能分支**: `###-aegis-knowledge-collector`
**创建时间**: 2025-05-02
**状态**: 草稿
**参考项目**: BiliNote（D:\AI\Athena\BiliNote）- 垂直 B 站视频笔记生成工具

**输入**: 用户描述: "项目：个人定制化知识库 —— 数据采集与笔记生成"
1. 总体目标
构建一个以 Obsidian 为核心阅读工具的本地知识库系统，第一阶段专注于从多平台收藏与书签中自动采集内容，并转换为结构化的 Markdown 笔记。生成格式参考 llm-wiki 风格（YAML 元数据、双链、标签），并由 Git 进行版本控制与历史追踪。

2. 核心数据来源（首批）
Bilibili（B 站）：用户收藏的视频（可能包括稍后再看、多个收藏夹）

知乎：收藏的回答、文章

稀土掘金：收藏的文章

CSDN：收藏的文章

浏览器书签：从导出的书签 HTML 或直接读取浏览器数据库，提取 URL 并抓取标题、简介等元数据，若指向视频/文章则按规则处理

后续将接入B 站已关注 UP 主的视频自动处理（订阅模式），但当前阶段先预留接口，主逻辑完成后再开发。

3. 内容处理规则
3.1 媒体类型识别
自动判断目标内容为视频还是纯图文（根据来源平台和 URL 特征）。

视频类（B 站视频为主）：

获取官方字幕（优先）或通过 ASR 生成字幕文本。

截图/封面作为笔记附件（可选保留）。

调用 AI 对字幕内容进行总结，生成更准确、结构化的文字说明，同时保留原始字幕全文。

图文类（知乎/掘金/CSDN/普通文章）：

直接抓取正文（包含代码块、图片链接），清洗广告、推荐等无关内容。

可选：AI 自动生成摘要（与视频类似，可配置开关）。

3.2 评论区处理
抓取所有评论（非仅热评），包括多级回复（尽量保留层级结构）。

可选择通过 AI 从评论区中提取关键信息、观点、补充知识，并整合到笔记中（作为"评论精华"章节）。

3.3 收藏夹与过滤
支持读取用户的多个收藏夹（如 B 站的默认收藏夹、自定义收藏夹）。

工作流：

拉取所有收藏夹列表，显示每个收藏夹名称及条目数。
用户选择需要处理的收藏夹（可多选、全选）。
系统列出该收藏夹下所有内容（标题、链接、添加时间），供用户手动过滤：勾选/剔除不需要生成笔记的条目（例如已失效、不感兴趣、重复）。尤其对于杂糅收藏夹，此步骤必须可交互。
确认后进入实际处理队列。
4. 增量处理与状态管理
维护一个本地处理清单（例如 SQLite 数据库或 JSON 文件），记录每一条已采集内容的：

来源平台 + 唯一 ID（如 B 站 cv 号、知乎回答 ID）

采集状态（待处理、处理中、已完成、失败、用户跳过）

最后更新时间（用于检测内容变更）

对应本地笔记文件路径

增量机制：

扫描收藏夹/书签时，先与处理清单对比，过滤掉已标记为"已完成"且原始内容未更新的条目。

用户手动跳过的条目记录状态"用户跳过"，除非手动重置，否则不再提示。

处理队列支持断点续传：可分批执行，下一次运行时继续处理未完成/待处理项。

本地笔记导入：

允许用户指定一个现有 Markdown 文件夹，系统扫描其中文件，提取基本信息（文件名、已有 YAML、标签），与处理清单关联。

为其自动补齐标准元数据字段（若无），并纳入 Git 版本控制，避免重复采集。

5. B 站失效与取消收藏
在处理过程中，若检测到视频已失效（如"已删除""视频不见了"），提供两种处理方式：

自动或手动调用 B 站 API ���消该收藏（需要 Cookie 授权）。
仅在笔记中标记"源已失效"，保留处理清单记录。
一键取消收藏功能：可单独触发，扫描全部收藏中失效内容并批量取消。

6. 笔记生成规范（llm-wiki 兼容 + Obsidian 优化）
每个收藏/书签生成 一个独立 .md 文件，文件名规则：[平台]-[ID]-[标题摘要].md（可配置）。

YAML Front Matter 至少包含：

yaml
---
title: "原始标题"
source: "Bilibili"
url: "原始链接"
author: " UP主/作者"
date: 2025-01-01
tags: [标签1, 标签2]
type: video/article
status: complete
---
正文结构建议（可模板化）：

AI 总结（如果生成）
原始内容全文（字幕 / 图文正文）
评论区（折叠块或引用）
相关链接（如作者主页）
自动添加 Obsidian 双链：针对作者、标签、来源平台等生成 [[作者]]、#标签，便于聚合。

参考 llm-wiki：尽可能使笔记独立、原子化，包含足够上下文，便于 AI 后期检索。

7. Git 版本控制
笔记根目录初始化为 Git 仓库，每次采集任务结束后自动执行 git add -A && git commit 并记录批注信息（如"采集 B 站收藏夹XXX，新增/更新 N 条"）。

支持配置 Git 远程仓库地址，用于多端同步。

8. 配置管理
所有可变参数集中到配置文件（如 config.yaml）：

平台认证（B 站 Cookie、知乎 Cookie、掘金等）

AI 后端选择（本地 / 远程 API / 无）

输出目录路径

笔记模板自定义

Git 自动提交开关

评论抓取深度（全部 / 热评前 N 条）

处理队列数据库路径

用户选择的要处理的收藏夹列表及过滤规则（可保存上次操作）

图片链接格式（支持配置，默认：相对路径 `./images/xxx.png`）

分流目录配置（根据内容属性设置，如 videos/、articles/、questions/）

**图片链接配置示例（config.yaml）**：
```yaml
image_format: "./images/{platform}/{id}/{filename}"  # 可配置的图片存储路径格式
```

**分流目录配置示例（config.yaml）**：
```yaml
routing:
  videos: "./notes/videos"      # B站视频
  articles: "./notes/articles" # 知乎/掘金/CSDN文章
  questions: "./notes/questions" # 知乎问答
```

9. 订阅模式（后续阶段，本阶段仅架构预留）
预留接口：获取已关注 UP 主列表，检查其最近视频，按时间增量产生处理项当前阶段只需在设计中考虑数据流可扩展，无需实现。"

## 与 BiliNote 的关系

Aegis 作为 Athena 的数据采集增强器，专注于：
- 多平台统一采集入口（CLI）
- 增量处理与状态管理
- llm-wiki 风格输出
- 成本控制（按需调用 AI）

BiliNote 提供核心能力：
- BilibiliDownloader：视频/音频下载
- 多种 Transcriber：字幕转写（Whisper/Groq/B-CUT 等）
- GPT 生成：笔记内容生成
